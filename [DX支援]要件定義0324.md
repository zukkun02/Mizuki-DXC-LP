# 要件定義書 — Notion中心のDX化コンサルティング LP

**作成日：** 2026年3月24日  
**バージョン：** v2.0（LP指示書反映済み）

---

## 1. プロジェクト概要

### 1.1 目的

Notion を中心とした DX 化コンサルティングサービスを訴求するランディングページ（LP）を制作する。  
単なる「Notion 構築代行」ではなく、**業務フローに合わせた一元管理の仕組みづくり＋定着支援まで一気通貫で提供できる** という差別化ポイントを明確に伝え、**無料ヒアリング申し込み（CV）** を最大化することが最大の目的。

### 1.2 販売商品（1本軸）

| 項目 | 内容 |
|------|------|
| 商品名 | Notion 中心の DX 化コンサルティング |
| 核心的な価値 | 「作って終わり」ではなく、**定着・運用まで支援する** |
| ターゲット | Notion を導入・活用したい企業（Notion 未経験でも可） |
| CV ゴール | 無料ヒアリング・相談の申し込み |

### 1.3 採用コピーライティングフレームワーク

**新 PASONA の法則**（お悩み解決型に最適）を全体構成に適用する。

| ステップ | 対応する LP セクション |
|----------|----------------------|
| P — Problem（問題提起） | ファーストビュー・課題共感セクション |
| A — Affinity（親近感） | 課題共感・代表者メッセージ |
| S — Solution（解決策） | サービス紹介・Why Notion |
| O — Offer（提案） | 無料ヒアリングの特典・サポート内容 |
| N — Narrowing Down（絞り込み） | 限定性・緊急性の訴求 |
| A — Action（行動） | CTA ボタン・お問い合わせフォーム |

## 2. ターゲットユーザー

### 2.1 メインターゲット（ペルソナ）

- デジタルツールを導入したものの社員に浸透しなかった経験がある企業担当者
- 業務のデジタル化（DX）を検討しているが何から始めればよいかわからない企業
- Notion を導入したい・検討中だが、社内に知見がない企業

### 2.2 ターゲットの課題（ペインポイント）

| No. | 課題 | 対応する心理テクニック |
|-----|------|----------------------|
| 1 | ツールを導入しても社員に使ってもらえず定着しない | 損失回避の法則 |
| 2 | 使える人と使えない人の格差が大きく、費用対効果が低い | 損失回避の法則 |
| 3 | 新規事業・業務変化に既存ツールが対応できず、追加コストが発生する | 損失回避の法則 |
| 4 | 導入・維持コストだけがかかり続けている | 損失回避の法則 |
| 5 | Notion の自由度が高すぎて「何をしたらいいかわからない」 | 親近感・共感 |

## 3. サービス内容・提供価値

### 3.1 提供サービス一覧

| # | サービス | 詳細 |
|---|---------|------|
| 1 | **徹底ヒアリング** | 複数回のヒアリングにより業務フローを把握。未経験でも「こんなことできる?」レベルから対応 |
| 2 | **要件定義** | ヒアリング内容をもとに御社専用の要件定義を策定 |
| 3 | **Notion 構築** | 業務フローに合わせた UI 最適化済みの Notion システムを構築 |
| 4 | **外部ツール連携** | Slack・GAS などの外部ツールとの連携も対応。Notion 単体に縛られない柔軟な構成 |
| 5 | **オーダーメイド映像マニュアル** | 御社専用の画面録画マニュアルを毎回手作りで提供 |
| 6 | **アフターサポート** | 納品後もフィードバック→修正のサイクルを継続。定着まで伴走 |

### 3.2 Notion を中心とする理由（差別化ポイント）

- 自由度が高く、**どんな業務フローにも柔軟に対応可能**
- 新規事業が発生してもページ追加で対応できる拡張性
- 外部ツール連携により、ほぼ全業務を Notion で一元管理できる時代になった
- ※ 数字管理（利益管理等）は Excel の方が適する場合もあるが、Notion でも対応可能

### 3.2.1 NPO向け追加広告でのヒアリング方針（追記）

- 無料ヒアリング時点で、対象業務を部分最適で切り分けず**全業務を対象に棚卸し**する
- 各業務ごとに「発生している課題」「現在の工数」「属人化ポイント」を明確化する
- その上で、Notion 導入時の改善余地を以下3軸で具体提案する  
  1) 業務効率化（手順整理・情報一元化）  
  2) 業務時間短縮（処理時間・確認時間の短縮）  
  3) 人的作業削減（転記・連絡・集計などの手作業削減）
- ヒアリング後の提案では、課題の洗い出し結果と Notion AI 活用案を紐づけて提示する
- 訴求メッセージは「作ること」ではなく、**業務課題の特定と解決設計**に置く

### 3.3 「続く仕組み」を実現するための取り組み

1. **UI の最適化** — 誰もが触りやすいインターフェース設計（自由度の高さを逆手に取り、迷わせない設計）
2. **オーダーメイドマニュアル** — 御社専用の動画マニュアルをいつでも見返せる形で提供
3. **フィードバックループ** — 構築後も打ち合わせ・修正を繰り返し、業務にフィットさせていく
4. **運用方法の提案** — 「誰がどう使うか」まで設計し、担当者が変わっても回る仕組みを構築

## 4. 実績・導入事例（社会的証明素材）

LP の「お客様の声 / 導入事例セクション（⑤）」に掲載する実績事例。  
**社会的証明・権威性** として活用し、「Notion でこんなことも実現できる」という具体的な納得感を与える。

### 事例①：GitHub × Notion 連携

| 項目 | 内容 |
|------|------|
| 業種・用途 | 開発チームのタスク管理 |
| 課題 | GitHub の Issue を Notion で一元管理したい。コメントやリアルタイム編集の反映も必要 |
| 実装内容 | GitHub と Notion を連携し、Issue の作成・コメント・編集がリアルタイムで Notion に反映される仕組みを構築 |
| LP での訴求ポイント | 外部ツール連携の実績／Notion の柔軟な拡張性 |

### 事例②：人材管理システム構築

| 項目 | 内容 |
|------|------|
| 業種・用途 | 採用・人材管理 |
| 課題 | 求職者の応募経路・面接通過率・内定率などの数字をひとつの場所で管理したい |
| 実装内容 | Notion の機能のみで人材管理システムを実装。応募経路別の通過率・内定率などを Notion データベースで可視化 |
| LP での訴求ポイント | Notion 単体での高度なデータ管理／数字の一元管理 |

### 事例③：LINE 友達リスト × Notion CRM 連携

| 項目 | 内容 |
|------|------|
| 業種・用途 | 顧客管理・CRM |
| 課題 | LINE の友達リストを Notion で管理し、顧客との接点をすべて一元化したい |
| 実装内容 | LINE 友達リストを Notion に取り込み CRM システムに組み込み。LINE でのアンケート結果も可視化し、すべてのアクションを Notion で管理できる状態を実現 |
| LP での訴求ポイント | 外部サービス（LINE）との連携実績／Notion を中心とした業務一元管理の具体例 |

### 4-0. 事例活用方針

- 各事例は LP の **導入事例セクション** で「課題 → 実装内容 → 成果」の形式で掲載する
- 可能であれば **ビフォーアフター形式**（管理が散在していた状態 → Notion 一元管理後の状態）で図解する
- 事例の多様性（開発・採用・CRM）を示すことで、**どの業種・業務でも対応可能** というメッセージを伝える

## 5. LP 構成要件（ページ設計）

LP は **ファーストビュー（FV）→ ボディ → クロージング** の3ブロック構成とする。  
各セクションに新 PASONA と7つの心理テクニックを意識的に埋め込む。

---

### 5.1 ファーストビュー（FV）

**役割：** 訪問後 3 秒で「自分に関係がある」と認識させ、離脱を防ぐ。

| 要素 | 内容 |
|------|------|
| メインキャッチコピー | **「ツールを入れても変わらない。それは、"仕組み"がないからです。」** |
| サブキャッチコピー | 「Notion を中心に、業務フローごとデジタル化。定着まで、伴走します。」 |
| 権威付け要素 | 導入実績・対応件数・メディア掲載（取得次第掲載） |
| CTA ボタン | 「まず無料で相談してみる」（緑 or オレンジ系。強いコントラスト） |
| マイクロコピー（CTA直下） | 「たった 3 分で入力完了 ／ 無理な営業は一切しません」 |
| 視線誘導 | **Z 型レイアウト**（左上ロゴ → 右上CTA → 左下キャッチ → 右下メインビジュアル） |
| 適用心理テクニック | 損失回避の法則・権威性 |

---

### 5.2 ボディ

#### ② 課題共感セクション（Problem ＋ Affinity）

**役割：** ペインポイントを言語化し「自分のことだ」と思わせる。

- 「こんなお悩みありませんか？」形式でペインポイント 5 項目を箇条書き
- 代表者の一言コメント or 顔写真付きのメッセージで**親近感**を演出
- 「私自身も同じ課題を目の当たりにしてきました」という共感の文章を入れる
- **適用心理テクニック：** 損失回避の法則・好意の法則（代表者の顔出し）

#### ③ 解決策セクション（Solution）

**役割：** 本サービスがペインの解決手段であることを論理的・感情的に伝える。

- 「なぜ Notion なのか？」を競合ツールとの比較表で示す（権威性・優位性）
- 「業務ツールに合わせるのではなく、業務フローに Notion を合わせる」という逆転の発想を訴求
- **適用心理テクニック：** 権威性・損失回避の法則

#### ④ サービス詳細セクション（Solution → Offer）

**役割：** 6 つのサービスのベネフィットと根拠（Evidence）を伝え、信頼感を醸成する。

- 各サービスをアイコン＋タイトル＋2〜3 行の説明で視覚的に整理
- **BEAF 構造** を意識：Benefit（得られる未来）→ Evidence（なぜ実現できるか）→ Advantage（競合との違い）→ Feature（具体的な機能）
- **適用心理テクニック：** 権威性・社会的証明（実績数値）

#### ⑤ 導入事例セクション

**役割：** 社会的証明で「自分でも効果が得られる」という確信を与える。  
**掲載事例：** セクション 4 で定義した3件の実績事例を使用する。

| 掲載順 | 事例タイトル | 訴求ポイント |
|--------|------------|------------|
| 1件目 | GitHub × Notion 連携 | 外部ツール連携の柔軟性 |
| 2件目 | 人材管理システム構築 | Notion 単体での高度なデータ管理 |
| 3件目 | LINE 友達リスト × Notion CRM | 業務一元管理の具体的成果 |

- 各事例は「課題 → 実装内容 → 成果」の形式で掲載
- ビフォーアフター形式（管理が散在していた状態 → Notion 一元管理後）で図解
- 3事例の業種の多様性（開発・採用・CRM）で「どんな業種・業務にも対応できる」ことを示す
- 顔写真または企業ロゴが取得できた場合は掲載（信憑性アップ）
- **適用心理テクニック：** 社会的証明・権威性

#### ⑥ 支援フロー（安心感の提供）

**役割：** 申し込みからサービス完了までの流れを示し、不安を取り除く。

- STEP 1：無料ヒアリング申し込み
- STEP 2：初回ヒアリング（業務フロー・課題ヒアリング）
- STEP 3：要件定義・提案書提出
- STEP 4：Notion 構築
- STEP 5：フィードバック・修正（複数回）
- STEP 6：納品・運用サポート開始
- **適用心理テクニック：** 一貫性の原理（小さな YES を積み重ねる）

#### ⑦ 返報性オファーセクション（Offer）

**役割：** 先に価値を提供し、行動のハードルを下げる。

- 「無料ヒアリング」だけでなく、**特典を明記**（例：業務フロー整理シートの無料プレゼント、初回相談でNotionサンプルページ提供 など）
- **適用心理テクニック：** 返報性の原理

---

### 5.3 クロージング

#### ⑧ FAQ セクション

**役割：** 購入直前の不安・疑問を解消する。

想定 FAQ 例：
- Notion を触ったことがなくても大丈夫ですか？
- 社員全員が使えるようになりますか？
- 構築にはどのくらいの期間がかかりますか？
- 料金はどのくらいですか？
- 既存の Excel・他ツールからの移行はできますか？

**適用心理テクニック：** 損失回避の法則（不安の先回り）

#### ⑨ 限定性・緊急性の訴求（Narrowing Down）

**役割：** 「今すぐ行動すべき理由」を示す。

- 月の受付件数 or 対応可能枠の上限を明示（例：「月 3 社限定受付」）
- 無料ヒアリング特典の提供期間を設定
- **適用心理テクニック：** 希少性の法則

#### ⑩ 最終 CTA ＋ フォーム（Action）

**役割：** 最終的な申し込みをスムーズに完了させる。

| 要素 | 内容 |
|------|------|
| CTA ボタン文言 | 「今すぐ無料でヒアリングを申し込む」 |
| マイクロコピー（直上） | 「今なら〇〇の特典付き ／ 無理な営業は一切しません」 |
| マイクロコピー（直下） | 「累計〇〇社が申し込み済み」（社会的証明） |
| フォーム項目（EFO） | 会社名・担当者名・メールアドレス・現在の課題（必須4項目のみ） |
| フォーム形式 | ステップ式（2ステップ以内）でリアルタイムバリデーション実装 |

---

### 5.4 CTA ボタンの配置（複数設置）

| 設置箇所 | 文言 |
|---------|------|
| ① FV 直下 | 「まず無料で相談してみる」 |
| ② ボディ中間（サービス詳細後） | 「御社の業務フローを一緒に整理しませんか？」 |
| ③ ページ最下部（フォーム直上） | 「今すぐ無料でヒアリングを申し込む」 |

#### 5.4.1 CTA種別・サイズ優先度（追記）

- メイン CTA は **無料相談ボタン** とし、最も視認性が高いサイズ・配色で設置する
- サブ CTA として **LINE 追加ボタン** を設置する（メイン CTA より小さいサイズ）
- LINE ボタンは「申し込み導線の近く」に配置し、独立したボタンとして視認可能にする
- SNS 導線は複数設置可能だが、**LINE 以外はアイコン表示を基本**とし、主導線を阻害しない
- CTA 階層は「無料相談 > LINE > その他 SNS アイコン」を厳守する

---

### 5.5 活用する7つの心理テクニック 対応表

| テクニック | 適用箇所 |
|-----------|---------|
| 社会的証明 | お客様の声・導入社数・CTA 直下マイクロコピー |
| 権威性 | FV の実績表示・サービス詳細の根拠 |
| 返報性の原理 | 無料ヒアリング特典・オーダーメイドマニュアル |
| 希少性の法則 | 月間受付件数の上限・限定期間特典 |
| 一貫性の原理 | 支援フロー STEP 表示・FAQ での小さな YES 積み重ね |
| 好意の法則 | 代表者の顔写真・個人のストーリー掲載 |
| 損失回避の法則 | 課題共感セクション・「このままでは〇〇になる」コピー |

## 5. 機能・技術要件

### 5.1 LP 技術要件

| 項目 | 要件 |
|------|------|
| デバイス対応 | PC・スマートフォン・タブレット（レスポンシブ対応） |
| お問い合わせフォーム | 会社名・担当者名・メール・現在の課題（4項目、EFO最適化） |
| フォーム形式 | ステップ式（2ステップ以内）、リアルタイムバリデーション |
| フォーム送信先 | メール通知 or Notion データベース連携（要確認） |
| SEO | タイトル・メタディスクリプション・OGP 設定 |
| アクセス解析 | Google Analytics または同等のツール導入 |

### 5.2 デザイン要件

| 項目 | 要件 |
|------|------|
| レイアウト | **Z 型レイアウト**（FV）／ **F 型レイアウト**（テキスト多いセクション） |
| 配色比率 | ベースカラー 70%・メインカラー 25%・アクセントカラー 5% |
| カラー | Notion ブランド（ブラック・ホワイト系）をベース＋アクセント（緑 or オレンジ）でCTA を際立たせる |
| フォント | 視認性の高い日本語フォント。見出しと本文に明確なジャンプ率をつける |
| 画像・アイコン | ビジネスシーン・Notion UI のスクリーンショット・代表者顔写真 |
| CTA ボタン | 背景と強いコントラストのある色（緑 or オレンジ）、ページ内 3 か所に設置。無料相談を最大サイズ、LINE は中サイズ、その他 SNS は小アイコン |
