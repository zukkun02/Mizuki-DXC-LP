### 5.2 業務棚卸しシート（メインの記入欄・最重要）

**目的**: 自動化対象になり得る業務をすべて洗い出す

#### 5.2.1 Notion DB 設計

| プロパティ名 | 型 | 必須 | 説明 / 入力例 |
|---|---|---|---|
| **業務名** | Title | ◯ | 例: 月初の請求書発行 / 候補者プロフィール要約 |
| **頻度** | Select | ◯ | 毎日 / 週次 / 隔週 / 月次 / 不定期 |
| **1回あたり時間（分）** | Number | ◯ | 例: 30 |
| **月合計時間（時間）** | Formula | 自動 | =頻度別係数 × 1回時間 / 60 |
| **関連ツール** | Multi-select | ◯ | Slack / Spreadsheet / Notion / メール / LINE / 紙 / その他 |
| **担当者** | Person/Text | △ | 自社内担当（フリーテキスト可） |
| **業務カテゴリ** | Select | ◯ | 転記・コピペ / 要約・サマリ / 連絡・通知 / 集計・分析 / 顧客対応 / ドキュメント生成 / 在庫・予約管理 / その他 |
| **どんな悩み？** | Text（短文） | △ | 例: 「毎回同じことの繰り返しで時間が取られる」 |
| **自動化できそう?** | Select | △ | はい / 一部 / 微妙 / わからない（→ 後で判定） |

#### 5.2.2 「月合計時間」の自動計算式（Notion Formula）
```
prop("頻度") == "毎日" → 1回時間×22 / 60
prop("頻度") == "週次" → 1回時間×4 / 60
prop("頻度") == "隔週" → 1回時間×2 / 60
prop("頻度") == "月次" → 1回時間×1 / 60
prop("頻度") == "不定期" → 1回時間×1 / 60（推測）
```

実装イメージ:
```
if(prop("頻度") == "毎日", prop("1回あたり時間（分）") * 22 / 60,
  if(prop("頻度") == "週次", prop("1回あたり時間（分）") * 4 / 60,
    if(prop("頻度") == "隔週", prop("1回あたり時間（分）") * 2 / 60,
      prop("1回あたり時間（分）") / 60
    )
  )
)
```

#### 5.2.3 初期データ（業種別テンプレに記入例3-5件）

業種別バリエーション（最低限の3例 + 業種ごと2例）:

**共通サンプル（全業種に表示）**:
1. 業務名: 顧客への定型メール返信 / 頻度: 毎日 / 1回時間: 20分 / カテゴリ: 連絡・通知 / ツール: メール
2. 業務名: 月次レポート作成 / 頻度: 月次 / 1回時間: 180分 / カテゴリ: 集計・分析 / ツール: Spreadsheet
3. 業務名: ミーティング議事録の整理 / 頻度: 週次 / 1回時間: 60分 / カテゴリ: 要約・サマリ / ツール: Notion

**業種別追加サンプル例**:
- **税理士事務所**: 月初の請求業務（月次/180分/集計分析）/ 顧問先からのヒアリング内容まとめ（週次/45分/要約）
- **人材エージェント**: 候補者プロフィール要約（毎日/30分/要約）/ 媒体別 KPI 集計（週次/120分/集計分析）
- **コンテンツクリエイター**: SNS 投稿の台本作成（週次/120分/ドキュメント生成）/ コメント返信（毎日/30分/連絡通知）
- **整体院**: 予約管理 + リマインド（毎日/15分/連絡通知）/ 問診票要約（毎日/15分/要約）
- **EC**: 問い合わせ対応（毎日/45分/顧客対応）/ 在庫レポート（週次/60分/集計分析）

3件はデフォルト表示、業種別追加分はユーザーが業種を選んだら自動表示する想定（Notion 単体では困難な場合、別ページに業種別テンプレ集として用意）。

#### 5.2.4 推奨入力数
- 最低 5件（最低限の判定が成立する数）
- 推奨 10〜15件（網羅性十分）
- 上限なし

ユーザーに「最低5件、できれば10件」と明記。

### 5.3 自動化レイヤー判定（業務 → 4層へのマッピング）

**目的**: 棚卸しした業務が「Notion / AI自動化 / エージェント / チーフAI」のどの層で削れるか、機械的に判定する

#### 5.3.1 判定ロジック（決定フローチャート）

```
業務カテゴリ
  ├─ 「転記・コピペ」「集計・分析」「在庫・予約管理」
  │     → ルールが明確 = Layer 1（Notion DB化）+ Layer 2（AI自動化）
  │
  ├─ 「要約・サマリ」「ドキュメント生成」
  │     → AIに任せられる = Layer 2（AI自動化）
  │
  ├─ 「顧客対応」「連絡・通知」
  │     → 一次対応はAI、判断必要部分は人 = Layer 2 + 3（自動化＋エージェント）
  │
  ├─ 複数ステップ判断必要（複雑な業務）
  │     → Layer 3（AIエージェント）
  │
  └─ 「その他」「わからない」
        → 無料相談で診断推奨
```

#### 5.3.2 Notion での実装方法

業務棚卸しテーブルに新たに **「推奨レイヤー」** プロパティ（Formula）を追加:

```
if(prop("業務カテゴリ") == "転記・コピペ", "Layer 1+2 (Notion+自動化)",
  if(prop("業務カテゴリ") == "集計・分析", "Layer 1+2 (Notion+自動化)",
    if(prop("業務カテゴリ") == "在庫・予約管理", "Layer 1+2 (Notion+自動化)",
      if(prop("業務カテゴリ") == "要約・サマリ", "Layer 2 (AI自動化)",
        if(prop("業務カテゴリ") == "ドキュメント生成", "Layer 2 (AI自動化)",
          if(prop("業務カテゴリ") == "連絡・通知", "Layer 2+3 (自動化+エージェント)",
            if(prop("業務カテゴリ") == "顧客対応", "Layer 2+3 (自動化+エージェント)",
              "要相談 (Layer 3 or 複合)"
            )
          )
        )
      )
    )
  )
)
```

#### 5.3.3 各レイヤーの解説（補足ページ）

ユーザーが「Layer 1って何？」と思ったときに見られる解説ページを用意:

**Layer 1: Notion 設計層**
- 役割: データを構造化して蓄積する基盤
- 具体例: 顧客DB、案件DB、議事録DB
- 効果: 散らばった情報が一箇所に集まり、AI が読める状態になる

**Layer 2: AI 自動化層**
- 役割: ルーティン業務を自動実行
- 具体例: メール内容の要約、議事録の整形、データ転記
- 効果: 月数十時間のルーティンが消える

**Layer 3: AI エージェント層**
- 役割: 業務単位で自律的に判断・実行
- 具体例: 顧客一次対応エージェント、営業フォローエージェント
- 効果: 「人がやらなくていい業務」が増える

**Layer 4: チーフ AI 層**
- 役割: 配下のエージェントを評価・改善するメタ層
- 具体例: 各エージェントのパフォーマンス分析、プロンプト最適化
- 効果: 使うほど賢くなる仕組み

各レイヤーに LP の Architecture セクションへのリンクを貼る。

### 5.4 削減時間 & ROI 試算（自動計算ダッシュボード）

**目的**: ユーザーが入力した業務から、削減見込み時間と人件費換算を自動表示

#### 5.4.1 集計ビュー（Notion 集計プロパティで実装）

| 指標 | 計算式 | 例 |
|---|---|---|
| **総業務時間（月）** | sum(月合計時間) | 75時間/月 |
| **自動化候補時間（月）** | sum(自動化できそう=はい+一部 の月合計時間) | 45時間/月 |
| **削減見込み時間（月）** | 自動化候補時間 × 0.66（控えめ係数） | 約30時間/月 |
| **削減見込み人件費（月）** | 削減時間 × 時給¥2,800（社保込み正社員平均） | ¥84,000/月 |
| **年間削減効果** | 月削減 × 12 | ¥1,008,000/年 |
| **Build Standard 投資回収期間** | ¥450,000 ÷ 月削減 | 約5.4ヶ月 |

#### 5.4.2 Notion の集計プロパティ実装

トップページに DB ビュー（テーブル）を埋め込み、`Sum` 集計で総業務時間を表示。
さらに別ページに「ROI ダッシュボード」を作り、以下を表示:

```
┌──────────────────────────────────────┐
│ あなたの削減ポテンシャル              │
├──────────────────────────────────────┤
│ 月の総業務時間          ─ X時間       │
│ うち 自動化できそう     ─ X時間（X%）│
│ 削減見込み時間（月）    ─ X時間       │
│                                      │
│ → 月の削減人件費        ¥XX,XXX      │
│ → 年間削減効果          ¥XXX,XXX     │
│ → Build Standard 回収   約X.X ヶ月    │
└──────────────────────────────────────┘
```

時給¥2,800 / Build Standard ¥450,000 中央値の前提を脚注で明示。

#### 5.4.3 計算前提の明示

ユーザーに「この数字はどう出してるの？」と思わせない透明性:
- 時給¥2,800 = 社保込み正社員平均時給（出典: 中小企業実態調査等）
- 削減係数 0.66 = 「自動化候補」のうち実際に削減可能な保守的見積もり
- Build Standard 中央値 ¥450,000 = LP 価格表より

### 5.5 優先順位マトリクス（効果 × 実装難易度）

**目的**: 「どこから着手すべきか」を視覚的に提示

#### 5.5.1 マトリクス設計

```
        効果（削減時間/月）大
              ▲
              │
   実装難 ←─┼─→ 実装易
              │
              ▼
        効果 小
```

業務を 4象限に分類:
- **右上（高効果・易実装）= 最優先**
- 左上（高効果・難実装）= 次に検討
- 右下（低効果・易実装）= サクッと着手
- 左下（低効果・難実装）= 後回し

#### 5.5.2 Notion での実装

業務棚卸しテーブルに新たに **「実装難易度」** プロパティ（Select: 易 / 中 / 難）を追加。
ユーザーが各業務に対して直感で選択。

別ページに **「優先順位ビュー」** を作成:
- グループ化: 推奨レイヤー × 実装難易度
- ソート: 月合計時間 降順
- ハイライト: 月合計時間 ≥ 5時間 かつ 実装難易度 = 易 → 「最優先」マーク

#### 5.5.3 トップ3 推奨業務の自動抽出

ROI ダッシュボードに「最優先で取り組むべき業務 TOP 3」を表示:
- 条件: 自動化候補 = はい/一部 + 月合計時間 上位3件
- 各業務に「推奨アクション」を併記

## 13. 開発時のFAQ（制作者向け）

**Q1. Notion Formula で複雑な集計が組めない場合は？**
→ Notion AI / 集計プロパティの組み合わせで対応。それでも厳しい場合は Google Sheets 版を別途用意。

**Q2. 業種別テンプレを別 DB にすべきか？**
→ 同一 DB に「業種」プロパティを持たせ、業種フィルタで切り替えるのが推奨。複数 DB はメンテ負荷が高い。

**Q3. PDF 版で計算が必要な場合は？**
→ 計算欄は空白にし、ユーザーが手計算する想定。Notion 版で計算済みの値を PDF にエクスポートする運用も可。

**Q4. 「自動化できそう?」の判定は誰が決める？**
→ ユーザーが直感で選ぶ。シートでは確定診断はせず、無料相談で精度の高い判断を行う設計。

**Q5. 個人情報は入力させる？**
→ 個人情報は入力させない（業務名のみ）。匿名で記入完了し、相談時に企業名や具体内容を共有する流れにする。
