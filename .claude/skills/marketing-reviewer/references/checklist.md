# マーケ観点チェックリスト

各項目: 現状を観察 → 課題を特定 → 改善提案

## A. ヒーロー(ファーストビュー)
- [ ] 5W1Hが1秒で分かる
- [ ] ターゲット解像度(誰の悩み?)が明示されている
- [ ] メインCTAが折り返し前に見える
- [ ] サブCTAとの主従関係が明確
- [ ] ノイズ要素(装飾/写真)がメインCTAを邪魔していない

## B. CTA配置と動線
- [ ] スクロール2〜3回に1つCTA
- [ ] CTAボタンの文言は具体的(「無料相談」より「30分Notion設計相談」)
- [ ] LINE と Jicoo の役割分化が一目で分かる
- [ ] 「次のセクションを読みたくなる」橋渡しがある
- [ ] スマホでタップ領域44×44px以上

## C. LINE誘導(計測なし、設計判定)
- [ ] LINE誘導の前に「LINEで何が得られるか」を明示
- [ ] 配信頻度・内容のサンプルが見える
- [ ] LINE登録の障壁(QR必要 vs ワンタップ)が低い
- [ ] LINEとメール・予約フォームが競合していない

## D. Social Proof
- [ ] 数字3指標の信頼性(出典・期間)が読み取れる
- [ ] 数字とお客様の声が同じスクロール範囲に並ぶ
- [ ] 顧客名イニシャル/属性で具体感がある
- [ ] スクリーンショットや動画で証拠補強

## E. 限定性・希少性
- [ ] 「月3社限定」が複数箇所で確認できる
- [ ] 残枠/締切日が動的(または最近更新)
- [ ] 限定性の理由が説明されている(品質維持等)

## F. 反論処理(FAQ)
- [ ] 「高そう」「自分にできる気がしない」「ツール変えても無駄」など主要不安を網羅
- [ ] FAQ後に必ずCTAがある

## G. ファイナル誘導
- [ ] 最終CTA前に決断を後押しする一言(限定性 / 保証)
- [ ] 最終CTAは画面下部に固定 or 大きく目立つ
- [ ] 「もう一度迷う場所」が無い

## H. ストーリー一貫性
- [ ] ヒーローの約束 → 中盤の証明 → 最終CTAの取引提案 が一直線
- [ ] トーン(です・ます調・落ち着き)が全体で一貫
