# lp-implementer ハードガード

## 絶対禁止リスト
1. マーカー区間 `<!-- LP_SYNC:NAME:start -->` 〜 `<!-- LP_SYNC:NAME:end -->` の外を編集
2. 既存のTailwind class名(`bg-primary`, `font-heading` 等)の書き換え
3. 新規 `<section>` の追加
4. `<head>` 内の編集
5. `<script>` 内の編集
6. ステータスが「承認」以外のカードへの操作
7. 「自動実装可=✗」のカードへの操作
8. `images/` ディレクトリ内のファイル変更
9. `vercel.json` / `.git/` の変更

## diff適用前チェック
- diff の追加行/削除行が マーカー区間内のみであること
- 削除行に `<!-- LP_SYNC:` を含まないこと
- 追加行に `<script>` `<link>` `<head>` を含まないこと

## 検証失敗パターンと対応

| パターン | 対応 |
|---|---|
| マーカー消失 | ロールバック→差し戻し |
| `<div>` カウント不整合 | ロールバック→差し戻し |
| ファイルサイズ ±30%超 | ロールバック→差し戻し+ユーザー警告 |
| UTF-8 BOM混入 | ロールバック→差し戻し |
| git add でstaging失敗 | ロールバック→ユーザーに権限確認依頼 |

## カード単位処理(全件一括禁止)
1カードごとに「適用→検証→commit準備→次へ」と進める。
全カードを書き換えてから検証する一括処理は **絶対に禁止**(失敗時の切り分け不能)。

## デプロイ前の最終確認
全カード処理後、`git diff --stat` で変更行数を表示し、
**もし300行を超える変更があった場合** はユーザー確認を待つ(暴走疑い)。
通常、テキスト/数字置換だけなら数十行程度で収まる。
